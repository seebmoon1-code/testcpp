⚡ وب سرور C++ پیشرفته (با SQLite و Multi-Thread)

پروژه‌ای که پایداری، سرعت و امنیت را در یکجا جمع کرده است.

معرفی و وضعیت سرور

این سرور با زبان **C++17** توسعه داده شده و از قابلیت‌های مدرن (مانند `std::thread`، `std::atomic` و `std::unique_ptr`) برای عملکرد بهینه استفاده می‌کند.

وضعیت: ✅ در حال اجرا (LISTEN) پورت پیش‌فرض: 8080 مدل همروندی: **Per-Request Threading** (چندنخی)

جزئیات فنی (Data Persistence)

ما از یک دیتابیس توکار و سبک به نام **SQLite3** استفاده می‌کنیم. این دیتابیس تمام داده‌های API را به صورت **پایدار** در فایل `server_db.sqlite` ذخیره می‌کند. حتی با ری‌استارت شدن سرور، اطلاعات از بین نمی‌رود.

دیتابیس **SQLite3** (سبک، توکار، بدون نیاز به سرور جداگانه)
فایل دیتابیس `server_db.sqlite`
جدول اصلی `users` (شامل ستون‌های `id`, `name`, `email`)
امنیت در کد استفاده از `db_connection_mutex` برای قفل کردن دسترسی به SQLite در محیط چندنخی

تست API (مدیریت کاربران - JSON CRUD)

این بخش برای تست مسیرهای API که با دیتابیس تعامل دارند، طراحی شده است.

مشاهده کاربران (GET: /api/users)

✅ هر بار که کاربر جدید اضافه می‌کنید، یک ردیف جدید در فایل `server_db.sqlite` ایجاد می‌شود.

ابزارهای سرور و امکانات اصلی

  • مدیریت فایل‌ها (/files)

    امکان **آپلود جریانی (Streaming Upload)** فایل‌های حجیم و سپس دانلود یا حذف آن‌ها. فایل‌ها در پوشه `uploads` ذخیره می‌شوند.

  • شمارنده بازدید (/count)

    تست Multi-thread بودن سرور: از یک `std::atomic` استفاده شده تا حتی با درخواست‌های همزمان، خطا یا رقابت بر سر داده‌ها (Race Condition) رخ ندهد.

راهنمای استفاده از سرور

  1. **کامپایل:** مطمئن شوید که کتابخانه‌های `g++` و `libsqlite3-dev` نصب شده‌اند. از دستور لینک‌دهی زیر استفاده کنید: 
    g++ -o server main.cpp -std=c++17 -pthread -lsqlite3
  2. **راه‌اندازی:** سرور را با `./server` اجرا کنید. در این حالت، فایل دیتابیس `server_db.sqlite` به طور خودکار ایجاد می‌شود.
  3. **تست API:** روی دکمه **"افزودن کاربر تستی"** کلیک کنید. یک کاربر در دیتابیس ذخیره شده و سپس لیست کاربران (API GET) در یک تب جدید باز می‌شود.
  4. **تست پایداری:** سرور را متوقف کرده و دوباره اجرا کنید. کاربران اضافه شده باید همچنان در `/api/users` قابل مشاهده باشند.
